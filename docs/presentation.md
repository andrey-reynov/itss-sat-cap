# Презентация

Полный набор схем для выбора и вставки в слайды находится в [diagrams.md](diagrams.md).

## Слайд 1. Тема и проблема

**Тема:** Разработка системы интеллектуального управления пропускной способностью спутникового канала связи на основе прогнозирования трафика.

Тезисы:

- спутниковый канал имеет ограниченную ёмкость;
- нагрузка во времени меняется неравномерно;
- статическое выделение полосы даёт либо перегрузки, либо избыточный резерв;
- нужна система, которая заранее оценивает будущую нагрузку и адаптирует ресурс.

## Слайд 2. Почему фиксированная ёмкость неэффективна

- при пиковом трафике фиксированная ёмкость не успевает за спросом;
- при спаде трафика часть ресурса простаивает;
- поэтому управление должно быть адаптивным;
- основой адаптации выбран прогноз трафика.

## Слайд 3. Архитектура системы

```mermaid
flowchart LR
    A["Данные CESNET"] --> B["Подготовка и признаки"]
    B --> C["Прогноз нагрузки"]
    C --> D["Управление ёмкостью"]
    D --> E["Оценка KPI"]
    E --> F["Сравнение с baseline"]
```

Ключевая фраза:

Прогноз используется не сам по себе, а как вход для модуля управления ресурсом.

## Слайд 4. Данные и модель канала

- использован открытый датасет CESNET-TimeSeries24 sample;
- выбрана почасовая агрегация `1 hour`;
- в качестве нагрузки используется суммарный `n_bytes`;
- это не прямые спутниковые измерения, а имитационная модель канала на основе открытого временного ряда.

## Слайд 5. Прогнозный модуль

- построены лаговые и календарные признаки;
- leakage исключён;
- сравнивались `NaiveLastValue`, `SeasonalNaive24`, `OLSRegression`;
- лучшей оказалась `NaiveLastValue`.

## Слайд 6. Модуль управления пропускной способностью

- baseline: фиксированная ёмкость канала;
- adaptive policy: ёмкость рассчитывается по прогнозу с резервным запасом;
- цель управления: снизить перегрузки и повысить долю обслуженного трафика.

Визуал:

- использовать график [capacity_control_plot.png](../results/capacity_control_plot.png).

## Слайд 7. Результаты

- `overload_rate`: `2.91%` -> `0.67%`;
- `served_traffic_ratio`: `99.23%` -> `99.90%`;
- суммарная перегрузка снижена примерно в `7.86` раза.

## Слайд 8. Выводы и развитие

- разработана рабочая имитационная система управления пропускной способностью;
- прогноз трафика дал полезный управляющий сигнал;
- адаптивное управление снизило перегрузки канала;
- работу можно развивать за счёт реальных спутниковых данных и более сложных методов управления.
