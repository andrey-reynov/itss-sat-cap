# Contributing

## Цель репозитория

Этот репозиторий предназначен как учебный шаблон для студентов по проекту ITSS Variant 9. Основной приоритет изменений:

- сохранять простую и понятную структуру;
- не добавлять тяжёлые файлы в git;
- не усложнять код без явной необходимости;
- держать документацию и ноутбук синхронизированными с кодом.

## Что можно улучшать

- формулировки в `README.md` и документации;
- визуализации и схемы в `docs/diagrams.md`;
- качество и читаемость ноутбука;
- политику управления ёмкостью;
- дополнительные эксперименты, если они не ломают базовый сценарий.

## Что не нужно коммитить

Не добавляйте в репозиторий:

- `datasets/ip_addresses_sample.tar`;
- большие промежуточные CSV из `results/`;
- локальные кэши и временные файлы;
- персональные пути, завязанные на локальную машину.

## Как вносить изменения

1. Установить зависимости из `requirements.txt`.
2. При необходимости скачать большой архив датасета отдельно.
3. Проверить, что `python3 satellite_capacity_project.py` запускается без правок путей.
4. Если меняется логика проекта, обновить:
   - `README.md`;
   - `docs/report.md`;
   - `docs/presentation.md`;
   - `docs/speech.md`, если изменились цифры или тезисы.

## Минимальная проверка перед коммитом

- проект запускается при наличии датасета;
- ноутбук и README не содержат ссылок на старую папку `ITSS`;
- в репозитории нет файлов больше лимита GitHub;
- reference-метрики и текстовые материалы не противоречат друг другу.
